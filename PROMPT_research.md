# Research Mode — Idea Feasibility Analysis

You are researching an idea from IDEAS.md to determine if it should be promoted to a formal spec.

## Your Task

1. Read the idea entry provided below
2. Answer the open questions listed
3. Assess technical feasibility within the project's constraints
4. Update the idea's status to `ready` (viable) or `rejected` (not viable)
5. Add your findings to the Description section

## Evaluation Criteria

- **Fits constraints**: Does it align with the project's technical stack and phase?
- **Clear scope**: Can it be broken into verifiable tasks?
- **Testable**: Can acceptance criteria be written as automated tests?
- **Value**: Does it solve a real problem or improve the system measurably?

## Output Format

Update IDEAS.md directly:
- Change status to `ready` or `rejected`
- Append findings to Description section under a "### Research Findings" heading
- Answer each open question inline or remove if resolved

Do not create spec files — that happens in promote mode.
